## 第一阶段：后端基础设施（优先）

### 1. 接入 Supabase 数据库

- [ ] 注册 Supabase，创建项目
- [ ] 获取 `DATABASE_URL` 连接串
- [ ] 执行 `npx prisma migrate dev` 初始化表结构
- [ ] 验证数据库连接正常

### 2. 部署到 Vercel

- [ ] 注册/登录 Vercel，连接 GitHub 仓库
- [ ] 配置环境变量：
  - `DATABASE_URL`（Supabase 连接串）
  - `AI_PROVIDER`（qwen）
  - `QWEN_API_KEY`（服务端 Key，用户无需填写）
  - `NEXTAUTH_SECRET`
  - `NEXTAUTH_URL`（Vercel 域名）
- [ ] 完成首次部署，验证 Web 版可访问

### 3. 打通用户注册登录

- [ ] 改造 NextAuth，支持邮箱+密码注册
- [ ] 创建 `/register` 注册页面
- [ ] 登录后跳转到 `/home`
- [ ] 保护需要登录的页面（middleware）
